Explore the granular components within each architectural layer to understand the specific roles and opportunities for Stripe and GSI Partners.

1. User Interaction Layer

Conversational UI/UX

The interface for user interaction (text, voice, image).

Stripe's Role: Provides `Stripe Elements` for embedding secure payment fields within any UI.

Partner Opportunity: Design and build the entire front-end user experience. Implement Natural Language Processing (NLP) and intent recognition.

User Intent Modeling

Translating a user's ambiguous request into a structured plan.

Stripe's Role: N/A

Partner Opportunity: Core GSI consulting and AI development role. Define user journey maps and prompt engineering strategies.

2. Agent Core

LLM & Reasoning Engine

The "brain" that processes user intent and plans tasks.

Stripe's Role: Model-agnostic. Provides an agent toolkit that works with any LLM that supports function calling (MCP).

Partner Opportunity: Foundational GSI role. Select, fine-tune, and host LLMs. Design and implement reasoning logic.

Agent Orchestration

Managing workflows between multiple specialized agents.

Stripe's Role: N/A. Toolkit plugs into orchestration frameworks.

Partner Opportunity: Select and implement frameworks like LangGraph or CrewAI. Design multi-agent architectures.

Memory & State Management

Maintaining context and retaining user preferences.

Stripe's Role: `Link` securely stores payment methods, shipping info, and email for recall.

Partner Opportunity: Implement agent memory modules. Integrate with vector databases and user profile stores.

3. Buy-Side Execution Layer

Product Discovery & Search

Agent's ability to search across multiple merchants.

Stripe's Role: N/A

Partner Opportunity: Implement web crawling, database queries, and API calls. Develop "AI SEO" (AEO) strategies.

Comparison & Recommendation

Analyzing products against user criteria to recommend.

Stripe's Role: Can enhance recommendations if a merchant shares data via ACP concept.

Partner Opportunity: Develop complex comparison logic, scoring algorithms, and personalization engines.

Programmatic Purchase

The final act of making the purchase from a merchant site.

Stripe's Role: `PurchaseIntent API` (preview) and `Issuing` for single-use virtual cards.

Partner Opportunity: Implement agent logic to call Stripe APIs, handle variant selection, fulfillment choices, and errors.

4. Sell-Side Enablement Layer

Product, Inventory & Order APIs

Allows agents to query products, check stock, and place orders.

Stripe's Role: Proposing "Agentic Commerce Protocol" (ACP) as an open spec for these APIs.

Partner Opportunity: CRITICAL GSI SERVICE. Design, build, and deploy these APIs for merchants. Modernize legacy PIM, IMS, and OMS systems.

Promotions & Pricing API

Allows agents to discover discounts or negotiate pricing.

Stripe's Role: ACP concept includes endpoints for promotions and negotiation.

Partner Opportunity: Build dynamic pricing and promotion logic into the merchant's backend and expose it via API.

The Future is Interoperable: The Protocol Landscape

A mature agentic economy will be built on open standards, not walled gardens. Understanding these complementary protocols is key to building future-proof solutions.

Agentic Commerce Protocol (ACP)

A Stripe-proposed spec for standardizing how merchants expose product, inventory, and order APIs to agents.

Agent2Agent (A2A) Protocol

An open protocol for enabling structured communication and collaboration between agents across platforms.

Model Context Protocol (MCP)

Standardizes how applications provide context and tools (like Stripe's APIs) to LLMs.

Know Your Agent (KYA)

A governance framework for verifying an agent's identity and authority, essential for trust and fraud prevention.
